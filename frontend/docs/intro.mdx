## Why Physical AI?

Physical AI combines:
1. **Embodied intelligence** - AI that interacts with the physical world
2. **Real-time decision making** - <10ms inference latency on edge devices
3. **Sim-to-real transfer** - Train in simulation, deploy on hardware
4. **Multimodal perception** - Vision + language + sensors

Traditional AI operates in digital environments (text, images, web). Physical AI must handle:
- Unpredictable real-world conditions
- Safety-critical requirements (robots can cause physical harm)
- Latency constraints (network delays are unacceptable)
- Limited computational resources (robots run on batteries)

## ROS 2: The Middleware Foundation

**ROS 2 (Robot Operating System 2)** is the communication middleware for robots. It provides:

- **Topic-based pub/sub** - Asynchronous sensor data streaming
- **Services** - Synchronous request/response (configuration, queries)
- **Actions** - Asynchronous long-running tasks with feedback (navigation, manipulation)
- **DDS middleware** - Distributed, reliable communication
- **Cross-platform** - Linux, Windows, macOS
- **Language support** - Python (rclpy), C++ (rclcpp)

### Simple ROS 2 Example

Here's a minimal publisher in Python (rclpy):

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class MinimalPublisher(Node):
    def __init__(self):
        super().__init__('minimal_publisher')
        self.publisher_ = self.create_publisher(String, 'topic', 10)
        self.timer = self.create_timer(1.0, self.timer_callback)
        self.i = 0

    def timer_callback(self):
        msg = String()
        msg.data = f'Hello ROS 2: {self.i}'
        self.publisher_.publish(msg)
        self.get_logger().info(f'Publishing: "{msg.data}"')
        self.i += 1

def main(args=None):
    rclpy.init(args=args)
    node = MinimalPublisher()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

**What This Does**:
1. Creates a ROS 2 node named `minimal_publisher`
2. Publishes "Hello ROS 2" messages to the `topic` topic every second
3. Logs each message to console
